# 마법 목록

상태: 레거시 프로토타입 아카이브  
최종 갱신: 2026-04-02  
섹션: 성장 시스템

## 범위

이 문서는 과거 프로토타입 단계에서 사용하던 액티브, 버프, 설치, 온앤오프 계열 마법 카탈로그를 보존하는 아카이브 문서다.

## 문서 상태

- 이 문서는 더 이상 최신 기획 기준 문서가 아니다.
- 최신 스킬 기획은 [skill_system_design.md](/Users/leesanghyun/git-projects/java-projects/old/dungeon_mage/docs/progression/rules/skill_system_design.md)를 우선한다.
- 실제 구현/asset/effect/레벨 스케일 반영 상태는 [skill_implementation_tracker.md](/Users/leesanghyun/git-projects/java-projects/old/dungeon_mage/docs/progression/trackers/skill_implementation_tracker.md)를 우선한다.
- 실제 코드 기준은 [current_runtime_baseline.md](/Users/leesanghyun/git-projects/java-projects/old/dungeon_mage/docs/implementation/baselines/current_runtime_baseline.md)와 대조한다.
- 이 문서는 legacy 명칭과 초기 프로토타입 컨셉을 확인할 때만 참고한다.

## 현재 런타임과 리워크 문서의 관계

| 구분 | 역할 | 우선 사용 시점 |
| --- | --- | --- |
| `spell_catalog.md` | 프로토타입 마법 카탈로그 아카이브 | legacy 명칭 확인, 과거 구조 대조 |

## 설계 원칙

- 각 서클마다 최소 1개 이상의 실전용 마법이 존재해야 합니다.
- 각 속성군은 최소 1개 이상의 주력 공격 또는 설치 마법을 가져야 합니다.
- 패시브 마스터리와 직접 피해 마법을 문서상 분리해 관리합니다.
- 이 문서는 패시브가 아닌 실전 운용 마법만 다룹니다.
- 버프는 약하지 않아야 하며, 순간 폭발력을 만들어내는 핵심 수단이어야 합니다.
- 버프의 자세한 중첩과 조합 규칙은 [buff_system.md](/Users/leesanghyun/git-projects/java-projects/old/dungeon_mage/docs/progression/rules/buff_system.md)에서 관리합니다.
- 실제 데이터 필드 구조와 `skill_id` 규칙은 [skill_data_schema.md](/Users/leesanghyun/git-projects/java-projects/old/dungeon_mage/docs/progression/schemas/skill_data_schema.md)에서 관리합니다.
